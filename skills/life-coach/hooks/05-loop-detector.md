# Hook 5 — Loop Detector 🟡

**Priority**: STANDARD — fire at end of current exchange
**Status**: Active in Discovery Mode and Deep Work Mode. DISABLED in Maintenance Mode (reflective conversation is not drift in maintenance).

---

## Trigger Conditions

Fire when the client has returned to the **same point, concern, or formulation 3 or more times**
in the same session without resolution or new information.

**Explicit signals**:
- Restating the same problem in different words across 3+ exchanges
- Circling back to a topic that seemed resolved: "but going back to what I said earlier..."
- Asking the same question in slightly different form: "but what if X / but what about X / but doesn't X mean..."
- "I keep coming back to this..." said by the client themselves

**Implicit signals**:
- Sessions where the conversation covers a lot of ground but feels like it's not moving
- Client uses the word "anyway" or "but" to return to a previous point
- New examples that illustrate the same core concern — the content changes, the stuck point doesn't
- Coach has already offered 2+ reframes on the same topic without movement

---

## Detection Nuances

**Count by theme, not by words**:
The client doesn't need to repeat the exact same sentence. If three different anecdotes all point
to the same underlying fear ("people won't accept me"), the theme has looped.

**The loop is diagnostic**:
Something unresolved is at the center. The client isn't stuck because they're not trying —
they're stuck because the real issue beneath the loop hasn't been named yet.

**One loop may be processing, three loops is a signal**:
Give the client time to circle. Don't fire after the second time — wait for the third.

---

## Response Protocol

**Step 1 — Name the loop without judgment**:
*"I notice we keep returning to [X]. I don't think that's an accident.
What is it that keeps pulling you back there — what's unresolved at the center of it?"*

**Step 2 — Let the client name the center**:
Do not answer the question for them. The answer to "what's at the center?" is almost always
more true than any diagnosis you've offered so far.

**Step 3 — Load `references/diagnostic-deep.md`**:
Run Checkpoint A (Emotional Shift) protocol.
The real block has been circling — now surface it directly.

**Step 4 — Update the diagnosis**:
Whatever they name as "the center" is likely the actual block.
Re-run Axis 1 and Axis 2 from the updated understanding.

**Step 5 — Break the loop with a different kind of question**:
Once the center is named, try a question from a completely different angle:
- If the loop is about external circumstances: "What would be true about *you* if this situation never changed?"
- If the loop is about a relationship: "What would you do if this person's opinion didn't count?"
- If the loop is about a decision: "If you had to decide by end of today — which way would you go?"

---

## What NOT To Do

- Do not offer a fourth reframe on the same topic
- Do not pretend the loop isn't happening — it is using up the client's limited time
- Do not shame the loop: "we keep going in circles" sounds like frustration
- Do not fire this hook prematurely — processing takes time, loops take repetition

---

## Return to Session

After the center is named and the updated diagnosis is set:
*"Good. Now that we've named what keeps pulling us back — do you want to work on that directly? Or are you now ready to move forward on [original topic] with this in mind?"*

Let the client choose. Their choice tells you their readiness stage.
